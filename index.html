Summary: Latitude vs. Weather Metrics

The purpose of this project is to assess weather changes relative to distance from the equator. This involved accessing data from the OpenWeatherMap API in order to assemble a dataset of over 500 cities. Based on the latitudinal location, we can further divide cities into Northern and Southern hemispheres.

After finalizing the dataset, we implemented the Python packages pandas and Matplotlib to generate plots comparing a variety of weather metrics (temperature, cloud cloverage, humidity, windspeed) vs latitude. Each graph is accompanied with a timestamp of the date weather measurements were recorded. Graphs are plotted using a heatmap to better represent the highs and lows of each metric. This site provides the source data, scatterplot analysis graphs as well as explanations and desscriptions of any trends and correlations observed.

There are several important considerations regarding analysis and interpretation of the dataset that should be taken into account. First, the data was collected from a single day of recording and may not reflect seasonal-related changes in weather patterns. Exclusion criteria was any city with a humidity level greater than 100. Finally, There appear to be significantly more datapoints for cities in the northern hemisphere compared to the southern hemisphere. This may bias or skew the data.
